Part 1.1 — List Functions

Part 1. Function and the doc syntax

Function is a block of code which runs when it is called.

You can think of the functions as “incantations” in the magic world. In the magical world, whenever you need to use a magic, you will use the “incantation” to make it happen. Similarly, in the programming world, whenever you need to let python does something for you, you call functions to make it happen. For example, we have seen how to use functions to let python import packages and modules, open files we wanted to process, print out information for inspection.

There are three types of functions in Python:

  • Built-in functions, such as help() to ask for help, min() to get the minimum value, print() to print an object to the terminal,… You can find an overview with more of these functions here.
  • User-Defined Functions (UDFs), which are functions that users create to help them out; And
  • Anonymous functions, which are also called lambda functions because they are not declared with the standard def keyword.

In this module, we will focus on built-in functions.

1.1. Functions associated with python list

In python, everything is an object. (For now, it is ok if you don’t understand the concept of object . Later in this semester, we will spend a whole week to learn about this concept.) Python list is an object, dictionary is another object. For functions associated with an object, we call them via the doc syntax.

Let's take python list as an example and take a closer look. As we have learned, list can contain certain values we assigned to it. In addition, a list object also has functions associate with it. We can call the functions via the doc-syntax to interact with data in the list.

Let's say I have a list contain ten different names.

In [1]:
boys = ['James','Kelly','Andy','Nick','Mike','Eric','Bob','Chris','Danny','Frank']

If I wanted to sort the names alphabetically, I can use the .sort() function associate with the list to do so.

In [2]:
boys.sort()
Out [2]:
boys = ['Andy','Bob','Chris','Danny','Eric','Frank','James','Kelly','Mike','Nick']

In some cases, we may wanted to know the specific position of a value. For instance, in the sorted list, where is the name "Frank" positioned at? The .index() function associated with list object can be useful to solve this.

In [3]:
boys.index('Frank')
Out [3]:
5

One thing we need to always be careful about is that python starts indexing from zero. In the above example, if you count the number of people from the beginning of the list until "Frank", the intuitive answer is 6. However, when python count the numbers, it starts from zero. i.e., Andy--0, Bob--1, Chris--2, and so on.

Sometimes we need to "pick" and then removes the element at the specific position using index. Think of the name list as a customers' order list in a fast food restaurant. As the programmer, I wanted to "pick" the customer on the top of the list, complete the order, and remove his name from the list. the .pop() function is an effective way to help us complete this task:

In [4]:
print(boys)
now_serving = boys.pop(0)
print(now_serving)
print(boys)
Out [4]:
['Andy','Bob','Chris','Danny','Eric','Frank','James','Kelly','Mike','Nick']
Andy
['Bob','Chris','Danny','Eric','Frank','James','Kelly','Mike','Nick']

In the above example, the .pop() function picked the name at position 0 (which is Andy) out of the list, assigned it to a new variable now_serving, and then removed it from the original list.

Suppose another customer is coming in, we wanted to add a new name to the existing list, we can use the .append() function.Suppose the new customer's name is "Peter" , to add him into the list, we can do:

In [5]:
print(boys)
boys.append('Peter')
print(boys)
Out [5]:
['Bob','Chris','Danny','Eric','Frank','James','Kelly','Mike','Nick']
['Bob','Chris','Danny','Eric','Frank','James','Kelly','Mike','Nick','Peter']

Sometimes we wanted to create a copy of the existing list to play with. The .copy() function associate with list object can help us do this.

In [4]:
new_list = boys.copy()
print (new_list)
Out [4]:
['Andy','Bob','Chris','Danny','Eric','Frank','James','Kelly','Mike','Nick']

If you have some previous programming experience before, you may wonder if the .copy() function is really necessary. Why don't just do this by assigning values? This is a potential issue that can cause programming mistakes. Let's look at the following example:

In [5]:
boys1 =  ['Andy','Bob','Chris','Danny','Eric','Frank','James','Kelly','Mike','Nick']
boys2 = boys1
boys1.remove('Andy')
print(boys1)
print(boys2)
Out [5]:
['Bob','Chris','Danny','Eric','Frank','James','Kelly','Mike','Nick']
 ['Bob','Chris','Danny','Eric','Frank','James','Kelly','Mike','Nick']

This is what just happened: although the two list boys1 and boys2 are different in names, but they bot point to the same, identical list (i.e., the list with 10 names). As soon as you remove a name using boys1.remove('Andy') , this command will change the content in both names of the list. If you only wanted to change one list, and keep they other unchanged, do the following:

In [6]:
boys1 =  ['Andy','Bob','Chris','Danny','Eric','Frank','James','Kelly','Mike','Nick']
boys2 = boys1.copy()
boys1.remove('Andy')
print(boys1)
print(boys2)
Out [6]:
['Bob','Chris','Danny','Eric','Frank','James','Kelly','Mike','Nick']
 ['Andy','Bob','Chris','Danny','Eric','Frank','James','Kelly','Mike','Nick']

Above are some examples of list methods. To see all the functions (and examples) associated with python list, please go to this website.
