Part 3.1 — Class

Part 3.1. Class

Let's try to define the blueprint for an animal-object.

Before we can build an object, we need first to design the "blueprint" for it. The blueprint is known as class. When define a class, we use the keyword class, followed by the name of the class, followed by a colon to start the first line. Under the header, we always use a special method __init__ to define what are the attributes. In this example, let's say the animal have two attributes: name and weight. In addition, we also put in a special variable self as a function parameter. The self indicates that every object we created using this blueprint will all have these attributes and methods. After this, let's define a method (i.e., functionality) that allow the object introduce itself. Remember, methods are essentially functions associated with the object. The way we define methods are exactly the same as we define functions.

In [1]:
class Animal:

   def __init__(self, name = 'Animal Doe', weight = 0.0):
        self.name = name
        self.weight = weight
    def introduce_me(self):
        print("Hello, let me introduce myself")
        print("My name is {}".format(self.name))
        print("My weight is {} pounds".format(self.weight))
        print("-------------------------------\n")

You can look at the class blueprint of the animal object using the help() function

In [2]:
help(Animal)
